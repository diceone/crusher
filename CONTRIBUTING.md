# Contributing to Crusher

Thank you for considering contributing to Crusher! This document outlines the process for contributing to this project.

## How to Contribute

1. **Fork the repository**
   - Create your own fork of the project

2. **Create a branch**
   - Create a branch for your feature or bug fix
   - Use a descriptive name for your branch (e.g., `feature/special-candies` or `fix/match-detection-bug`)

3. **Make your changes**
   - Implement your feature or bug fix
   - Keep your changes focused and related to the branch purpose

4. **Test your changes**
   - Make sure your changes work as expected
   - Test the game in different browsers if possible

5. **Commit your changes**
   - Use clear and descriptive commit messages
   - Reference issue numbers if applicable

6. **Submit a pull request**
   - Provide a clear description of the changes
   - Link any related issues

## Code Style Guidelines

- Use consistent indentation (2 spaces)
- Follow JavaScript best practices
- Comment your code when necessary
- Keep functions small and focused

## Feature Suggestions

If you have ideas for new features:
1. Check if the feature has already been suggested
2. Open an issue describing the feature and its benefits

## Bug Reports

When reporting bugs:
1. Describe the bug clearly
2. Provide steps to reproduce
3. Include browser and OS information
4. Add screenshots if helpful
